# 部门管理导出即模板设计

## 背景

当前部门管理页支持模板导入，但不支持导出全部部门。现有导入接口只识别三列：`部门编号`、`部门名称`、`上级部门编号`。如果新增一个仅用于查看的导出格式，用户导出后仍然不能直接修改并重新导入。

本次目标是让“导出全部部门”生成的文件可以直接作为导入文件使用，实现“导出后修改，再直接导入”。

## 设计结论

采用单一可导入结构：

- 模板下载与全部导出使用同一套表头
- 表头固定为：`部门编号`、`部门名称`、`上级部门编号`
- 导出文件填充当前全部部门真实数据
- 模板下载保留示例数据，但结构与导出完全一致

不额外加入 `ID`、`锁定`、`上级部门名称` 等展示字段，避免导出文件和导入格式分叉。

## 变更范围

### 1. 页面入口

在部门管理页工具栏新增“导出全部部门”按钮，位置与“刷新”等操作保持同一层级。

### 2. 前端行为

新增按钮点击事件，直接跳转 `/admin/departments/export` 触发下载，不增加额外筛选或批量选择逻辑。

### 3. 后端接口

新增 `/admin/departments/export`：

- 权限与部门管理页保持一致，沿用管理员权限控制
- 使用 `openpyxl` 生成 `xlsx`
- sheet 表头为：`部门编号`、`部门名称`、`上级部门编号`
- 数据范围为全部部门
- 每行父级部门为空时导出空字符串
- 排序保持稳定，按 `dept_no`、`dept_name` 升序导出

### 4. 模板同步

保留 `/admin/departments/template` 下载入口，但让模板与导出共享同一套列定义，避免后续维护时再次不一致。

## 非目标

- 不修改部门导入的核心业务规则
- 不增加 CSV、筛选导出、勾选导出
- 不把锁定状态等不可导入字段塞进导出文件

## 测试与验证

新增后端测试覆盖：

1. 模板下载表头为 `部门编号`、`部门名称`、`上级部门编号`
2. 全部部门导出接口返回 200，且文件可被 `openpyxl` 正常打开
3. 导出文件包含父子部门数据时，`上级部门编号` 正确写出
4. 导出文件结构与导入接口一致，能支持“导出后修改再导入”的使用方式

## 风险与约束

导出格式优先服务“可回导”，因此可读性会弱于纯展示型报表。这是本次有意选择的取舍，目的是让用户只维护一种文件格式，减少误用。
